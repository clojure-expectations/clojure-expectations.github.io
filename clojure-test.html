clojure.test compatibility

Experimental clojure.test compatibility is available as part of the 2.2.0-SNAPSHOT release. There is a new expectations.clojure.test namespace containing a defexpect that allows for named expectations to be defined, like clojure.test's deftest macro.

(ns example.test
  (:require [expectations :refer :all]
            [expectations.clojure.test :refer [defexpect]]))

;; a single named expectation
(defexpect two 2 (+ 1 1 ))

;; a named group of expectations
(defexpect group
  (expect [1 2] (conj [] 1 2))
  (expect #{1 2} (conj #{} 1 2))
  (expect {1 2} (assoc {} 1 2)))

Expectations defined this way can be run with regular clojure.test tooling, such as lein test.
